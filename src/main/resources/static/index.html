Road network (shapefile)

Upload the road network shapefile zip. Choose whether to add/update roads or replace the whole network.

Mode
Shapefile zip (.shp/.shx/.dbf/.prj together) or GeoJSON

The file must carry a Section_La for every road. Validation runs first — if it fails, nothing is changed. After upload, click Build segments to re-cut condition data against the updated network.

Full Road Network (by Road Name)

Upload the full road-network shapefile zip — the second road layer in the viewer (no Section Label). Stored permanently in the database, so it survives refresh and restart. Roads are matched by Road_id (then road number, then name).

Mode
Shapefile zip (.shp/.shx/.dbf/.prj together) or GeoJSON

Re-importing a road with the same Road_id updates it; new ones are added. Toggle Full Road Network on in the viewer to see it.

Structures & road furniture

Upload assets as CSV — they are linearly referenced onto the network. Bridges and furniture lines need Section_Label, Start_Chainage, End_Chainage; culverts and furniture points need Section_Label, Chainage. Every other column is kept and shown in the popup.

Asset type
CSV file

Re-uploading a type replaces that type's records. Rows with missing chainage or an unknown Section_Label are skipped and reported.

Pavement & geotechnical data

Upload test data as CSV — placed on the network by chainage (FWD uses its own latitude/longitude when present). Needs Section_Label (or Label) and Chainage (FWD: From). Every other column appears in the popup.

Data type
CSV file

Re-uploading a type replaces that type's records. In the viewer, switch each layer on under Layers (Sub-Grade Soil, Bituminous Core, Pavement Crust, FWD) and click any icon for full details.

Traffic data

Import the traffic survey CSVs. Stations are placed on the road by chainage in the viewer; counts are summarised per station for the popup (ADT, peak hour, direction-wise). Held in this browser for the viewer.

1 · Traffic stations CSV
2 · Traffic count CSV
